# QA Feedback: Docker Compose Article

## Общие впечатления
Статья отлично структурирована и последовательно вводит в тему Docker Compose. Переход от проблем ручного управления к автоматизации через манифесты понятен. Однако есть несколько моментов, которые могут вызвать вопросы у новичка (Junior QA/DevOps).

## Замечания и вопросы

1. **Синтаксис команд (docker-compose vs docker compose):**
   * В статье упоминается Docker Compose V2, но во всех примерах используется `docker-compose` (с дефисом). В современных версиях Docker (Docker Desktop, новые дистрибутивы Linux) стандартом является `docker compose` (пробел вместо дефиса). Для новичка может быть непонятно, есть ли разница и какую команду использовать, если `docker-compose` не найдется в системе.

2. **Несоответствие в примере `build` и `Dockerfile`:**
   * В разделе "Анатомия" в блоке `build` указан параметр `args: NODE_ENV=production`. Однако в представленном далее `Dockerfile` для Flask-приложения отсутствует инструкция `ARG NODE_ENV`. Новичок может не понять, как именно эта переменная попадает в процесс сборки и зачем она там нужна, если она не используется.

3. **Путаница с "горячей перезагрузкой" (Hot Reload):**
   * В описании **Bind mounts** приводится пример с `nginx.conf` и утверждается, что это "основа процесса разработки (горячая перезагрузка)". Это технически не совсем верно для Nginx: он не подхватывает изменения конфига "на лету" без команды `reload`. Для новичка лучше привести пример с кодом приложения (например, Flask в debug-режиме), который действительно перезапускается сам при изменении файлов.

4. **Пути к Named Volumes:**
   * Указано, что именованные тома хранятся в `/var/lib/docker/volumes`. Стоит уточнить, что это актуально только для Linux. Те, кто использует Docker Desktop на Windows или macOS, не найдут эту папку напрямую, так как Docker работает внутри виртуальной машины.

5. **Команда `--scale`:**
   * В разделе CLI команда `--scale flask=3` указана просто списком. Новичок может попытаться выполнить её как самостоятельную команду. Было бы понятнее написать полный пример: `docker-compose up -d --scale flask=3`.

6. **Библиотеки Python (`psycopg2` vs `psycopg2-binary`):**
   * В тексте статьи говорится: "используем библиотеку `psycopg2`", но в `requirements.txt` и `Dockerfile` указана `psycopg2-binary`. Для новичка это разные сущности. Стоит придерживаться одного названия или пояснить разницу (что для Docker-образов на alpine/slim удобнее использовать binary-версию).

7. **Зачем нужен `healthcheck` в `depends_on`?**
   * **Вердикт:** Объяснено хорошо. Четко прописана разница между "контейнер запущен" и "сервис готов". Пример с "Connection Refused" очень наглядный и понятный.

8. **Опечатки в CLI:**
   * Явных опечаток в самих командах не обнаружено, синтаксис YAML в примерах корректен.

## Итог
Статья качественная, но требует небольшого уточнения в части соответствия примеров кода (build args) и актуализации синтаксиса команд для V2.
