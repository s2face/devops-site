# Отзыв Junior QA по статье "GitHub Actions: полный CI/CD пайплайн за один урок"

## Общее впечатление
Статья охватывает широкий спектр тем и дает хороший обзор возможностей GHA. Примеры кода структурированы и снабжены комментариями. Однако для полного новичка есть несколько "слепых зон", которые могут привести к ошибкам при попытке повторить шаги.

## Список замечаний и предложений

### 1. Отсутствие информации о расположении файлов
**Проблема:** В статье сразу дается YAML-код, но не говорится, где его создать.
**Предложение:** Добавить уточнение, что файлы воркфлоу должны лежать в папке `.github/workflows/` в корне репозитория.

### 2. Работа с GitHub Secrets
**Проблема:** В разделе деплоя используются `${{ secrets.SERVER_HOST }}` и другие. Студент может не знать, где их прописывать.
**Предложение:** Добавить краткую навигацию: `Settings -> Secrets and variables -> Actions -> New repository secret`. Также стоит пояснить, что `GITHUB_TOKEN` генерируется автоматически и его создавать не нужно.

### 3. Необходимые файлы в репозитории
**Проблема:** Пайплайн упадет на первом же шаге, если в репозитории нет `requirements.txt` или Dockerfile.
**Предложение:** В начале практической части добавить список файлов ("Checklist"), которые должны быть в репозитории, чтобы код из статьи заработал.

### 4. Хрупкость команды деплоя
**Проблема:** Команда `sed -i "s|image:.*|...|g" docker-compose.yml` предполагает определенную структуру файла. Если в `docker-compose.yml` несколько сервисов или другое форматирование, она может перезаписать лишнее или не сработать.
**Предложение:** Дать пример минимально необходимого `docker-compose.yml`, который совместим с этой командой `sed`.

### 5. SSH и безопасность
**Проблема:** Для новичка может быть неочевидно, какой именно ключ (приватный или публичный) класть в `SSH_PRIVATE_KEY`.
**Предложение:** Уточнить, что в секреты GitHub кладется **приватный** ключ, а **публичный** должен быть предварительно добавлен в `~/.ssh/authorized_keys` на целевом сервере.

### 6. Использование Trivy
**Проблема:** Параметр `exit-code: '1'` сразу остановит обучение студента, если в базовом образе Python найдутся уязвимости (а они там часто есть).
**Предложение:** Посоветовать для начала поставить `exit-code: '0'`, чтобы просто увидеть отчет в логах, не блокируя выполнение пайплайна.

### 7. Локальный запуск (Бонус)
**Предложение:** Было бы полезно упомянуть инструмент `act`, который позволяет запускать GHA локально для отладки, чтобы не тратить лимиты минут GitHub.

## Резюме
Статья отличная для среднего уровня, но для "Junior" уровня требует чуть больше пояснений по интерфейсу GitHub и предварительной подготовке окружения.
