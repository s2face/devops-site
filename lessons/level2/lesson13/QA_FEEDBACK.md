# QA Feedback: Урок 13 (Jenkins)
**Статус:** Требуются уточнения (Major issues found)
**Проверяющий:** @junior_qa

## 1. Критические замечания (Блокеры)
*   **Docker CLI внутри Jenkins:** В разделе про `docker-compose.yml` утверждается, что проброс сокета позволит запускать команды `docker build`. Это не так. В официальном образе `jenkins/jenkins:lts` нет исполняемого файла `docker`. 
    *   *Последствие:* Ученик получит `command not found` на этапе сборки образа.
    *   *Рекомендация:* Добавить пример простого `Dockerfile`, который базируется на Jenkins и устанавливает `docker-ce-cli`.

## 2. Логические ошибки в коде
*   **Использование Credentials:** В примере пайплайна для Docker в блоке `environment` мы инициализируем `DOCKER_CREDS`, но в шаге `Docker Push` используем `withRegistry('', 'docker-hub-id')`. 
    *   *Вопрос:* Зачем нам `DOCKER_CREDS` в окружении, если метод `withRegistry` сам лезет в Credentials Store по ID? Это путает.
*   **Область видимости:** Переменная `appImage` в одном стейдже создается, в другом используется. Для Junior-разработчика стоит пояснить, что отсутствие `def` делает её глобальной в рамках скрипта.

## 3. Недостаток пояснений (Terminology & Setup)
*   **SCM:** Поясните, что `checkout scm` — это "скачай код того репозитория, где лежит этот Jenkinsfile".
*   **Shared Libraries:** Нужно добавить примечание, что библиотеку мало создать в Git, её нужно прописать в настройках Jenkins (Manage Jenkins -> System -> Global Pipeline Libraries). Без этого `@Library` не заработает.
*   **Термины:** Не расшифрованы `Artifact`, `SaaS`, `Agent`.
*   **Blue Ocean:** Стоит упомянуть, что этот плагин постепенно устаревает (maintenance mode), хотя для визуализации он всё еще хорош.

## 4. Мелкие недочеты и опечатки
*   В тексте упоминается `cleanWs()`, но в примерах кода этот важный шаг очистки пропущен.
*   В `docker-compose.yml` указано `user: root`. Стоит добавить предупреждение (Warning), что это допустимо только для обучения, но не для продакшена.

## Заключение
Материал структурирован отлично, но требует "допила" в части запуска Docker внутри Docker. Без исправления инструкции по запуску, новичок не сможет пройти урок до конца.
