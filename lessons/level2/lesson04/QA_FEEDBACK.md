# QA_FEEDBACK.md

## Отзыв от @junior_qa (Уровень: Новичок в K8s)

### Общее впечатление
Статья хорошо структурирована и понятно объясняет теорию. Однако есть несколько практических моментов, которые вызвали затруднения.

### Замечания и вопросы:
1.  **Где установка самого minikube?** В разделе "Установка minikube" есть Шаг 1 (установка kubectl) и Шаг 2 (запуск minikube). Но нигде не сказано, как скачать и установить сам бинарный файл `minikube`. Без него команда `minikube start` не сработает.
2.  **Как увидеть работающий nginx?** Мы запустили Под (`kubectl run nginx`), проверили статус и логи. Но как мне, как пользователю, зайти на страницу "Welcome to nginx"? В Docker Compose я бы пробросил порты. В статье не хватает упоминания `kubectl port-forward` или хотя бы объяснения, почему мы пока не можем зайти на него через браузер.
3.  **Специфичность команд:** Команды установки `kubectl` указаны только для Linux (amd64). Стоит добавить примечание для пользователей macOS и Windows или дать ссылку на официальную документацию.
4.  **Аналогии для компонентов:** Понятия "Pod" и "kubelet" звучат абстрактно. Было бы круто добавить сравнение с Docker Compose (например, Pod как группа контейнеров из одного `docker-compose.yml`, а kubelet как локальный менеджер).
5.  **Что такое etcd?** Сказано, что это хранилище "ключ-значение", но новичку может быть непонятно, что именно там хранится (конфиги? данные базы данных приложения? состояние кластера?).

### Предложение:
Добавить раздел про `port-forward`, чтобы читатель мог пощупать результат своей работы.
