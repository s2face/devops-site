# Урок 13 — Введение в Docker: контейнеры vs VM

## Docker для начинающих: контейнеры vs виртуальные машины

В мире разработки и эксплуатации ПО долгое время доминировала проблема: «На моем компьютере всё работает, а на сервере — нет». Разные версии библиотек, несовпадающие системные зависимости и отличия в конфигурациях ОС превращали деплой в кошмар. Docker пришел, чтобы решить эту проблему раз и навсегда.

### Проблема «у меня работает» — и как Docker её решает

Раньше для изоляции приложений использовали Виртуальные Машины (VM). Однако VM тяжеловесны: каждая требует целую гостевую ОС со своим ядром, что потребляет гигабайты памяти и минуты на загрузку.

Docker предлагает другой подход — **контейнеризацию**. Вместо того чтобы виртуализировать всё железо, Docker изолирует процессы на уровне ядра основной (хостовой) ОС. Это позволяет запускать приложения в идентичном окружении где угодно: от ноутбука разработчика до облачного кластера.

## VM vs Container: в чём разница?

Главное различие кроется в архитектуре.

### Сравнение архитектур

**Виртуальные машины (VM):**
```text
[ Приложение А ] [ Приложение Б ]
[ Библиотеки   ] [ Библиотеки   ]
[ Гостевая ОС  ] [ Гостевая ОС  ]  <-- Жирный слой!
[      Гипервизор (Type 1/2)    ]
[        Хостовая ОС            ]
[         Железо                ]
```

**Контейнеры (Docker):**
```text
[ Приложение А ] [ Приложение Б ]
[ Библиотеки   ] [ Библиотеки   ]
[       Docker Engine           ]  <-- Общее ядро хоста
[        Хостовая ОС            ]
[         Железо                ]
```

### Сравнительная таблица

| Характеристика | Виртуальные машины (VM) | Контейнеры (Docker) |
| :--- | :--- | :--- |
| **Изоляция** | Полная (на уровне железа/ядра) | Процессная (на уровне пространств имен ядра Linux) |
| **Операционная система** | Каждая VM имеет свою гостевую ОС | Все контейнеры делят ядро хоста |
| **Скорость запуска** | Минуты | Секунды или миллисекунды |
| **Потребление ресурсов** | Высокое (RAM/CPU зарезервированы) | Низкое (используется столько, сколько нужно) |
| **Размер образа** | Гигабайты (ГБ) | Мегабайты (МБ) |

## Основные концепции Docker

Чтобы уверенно работать с Docker, нужно различать четыре сущности:

1.  **Образ (Image):** Неизменяемый шаблон (чертеж), содержащий приложение и всё необходимое для его запуска (код, библиотеки, конфиги).
2.  **Контейнер (Container):** Запущенный экземпляр образа. Это изолированный процесс.
3.  **Слои (Layers):** Образы состоят из слоев. Если вы меняете только одну строчку кода, Docker пересоберет только верхний слой, что делает сборку мгновенной.
4.  **Registry (Реестр):** Хранилище образов. Самый популярный — [Docker Hub](https://hub.docker.com).

## Установка Docker на Ubuntu 22.04

Самый быстрый способ установить Docker на Ubuntu — использовать официальный скрипт от Docker.

### 1. Установка через скрипт
```bash
# Скачиваем и запускаем установочный скрипт
curl -fsSL https://get.docker.com | sudo sh
```

### 2. Настройка прав (Post-install)
Чтобы не писать `sudo` перед каждой командой `docker`, добавьте своего пользователя в группу `docker`:
```bash
sudo usermod -aG docker $USER
```
*Важно: После этого нужно перезайти в систему или выполнить `newgrp docker`, чтобы изменения вступили в силу.*

### 3. Проверка версии
```bash
docker --version
# Должно быть Docker version 24.x.x или выше
```

## Первые шаги: запускаем контейнер

### Hello World
Самый простой тест — запуск образа, который просто выводит текст в консоль:
```bash
docker run hello-world
```
Docker проверит, есть ли образ локально. Если нет — скачает его с Docker Hub, создаст контейнер, запустит его и выведет приветствие.

### Запуск веб-сервера Nginx
Давайте запустим что-то более полезное — полноценный веб-сервер.
```bash
docker run -d -p 8080:80 --name my-web nginx
```

**Разбор команды:**
*   `docker run` — создать и запустить контейнер.
*   `-d` (detached) — запустить в фоновом режиме.
*   `-p 8080:80` — проброс портов (Port mapping). Порт 8080 на вашем хосте будет вести на порт 80 внутри контейнера.
*   `--name my-web` — дать контейнеру понятное имя (вместо случайного).
*   `nginx` — название образа.

Теперь откройте браузер и перейдите по адресу `http://localhost:8080`. Вы увидите стандартную страницу "Welcome to nginx!".

## Разбор ключевых концепций: Сеть и Данные

### 1. Port Mapping (Проброс портов)
Контейнеры живут в своей изолированной сети. Чтобы попасть внутрь контейнера извне (например, из браузера), нужно "открыть дверь".
Синтаксис: `-p <PORT_ON_HOST>:<PORT_IN_CONTAINER>`

Пример: `-p 3000:80` — приложение внутри контейнера слушает 80 порт, но мы заходим на него через 3000 порт нашего сервера.

### 2. Volumes (Тома)
Контейнеры **эфемерны**. Это значит, что если вы удалите контейнер, все данные, созданные внутри него, исчезнут. Чтобы сохранить данные (например, базу данных или логи), используются Volumes.
Синтаксис: `-v <HOST_PATH>:<CONTAINER_PATH>`

Пример:
```bash
docker run -d -v /home/user/html:/usr/share/nginx/html -p 80:80 nginx
```
Теперь файлы из папки `/home/user/html` на вашем компьютере будут отображаться внутри nginx. Если вы измените файл на хосте, изменения мгновенно появятся на сайте.

### 3. Environment Variables (Переменные окружения)
Часто приложению нужно передать настройки (пароль к БД, API ключ) без изменения кода. Для этого используется флаг `-e`.
Пример:
```bash
docker run -e MYSQL_ROOT_PASSWORD=my-secret-pw -d mysql
```

## Таблица: Полезные флаги `docker run`

| Флаг | Полное имя | Описание | Пример |
| :--- | :--- | :--- | :--- |
| `-d` | `--detach` | Запуск в фоновом режиме | `docker run -d nginx` |
| `-p` | `--publish` | Проброс портов | `-p 8080:80` |
| `--name` | `--name` | Имя контейнера | `--name my-app` |
| `-v` | `--volume` | Монтирование папок/томов | `-v /data:/app/data` |
| `-e` | `--env` | Переменные окружения | `-e DEBUG=true` |
| `-it` | `--interactive --tty` | Интерактивный режим (для терминала) | `docker run -it ubuntu bash` |
| `--rm` | `--rm` | Удалить контейнер после остановки | `docker run --rm alpine ls` |

## Команды управления

### Работа с контейнерами
*   `docker ps` — список запущенных контейнеров.
*   `docker ps -a` — список всех контейнеров (включая остановленные).
*   `docker stop <ID_или_имя>` — остановить контейнер.
*   `docker start <ID_или_имя>` — запустить остановленный контейнер.
*   `docker rm <ID_или_имя>` — удалить контейнер.
*   `docker logs -f <ID_или_имя>` — смотреть логи контейнера в реальном времени.
*   `docker exec -it <ID_или_имя> bash` — "войти" внутрь запущенного контейнера.

### Работа с образами
*   `docker images` — список скачанных образов.
*   `docker pull <image>` — скачать образ с Docker Hub без запуска.
*   `docker rmi <image_id>` — удалить образ.
