## Прежде чем начать: Словарь новичка

Linux-сообщество говорит на своем языке. Чтобы вы не чувствовали себя иностранцем в чужой стране, давайте разберем основные понятия, которые встретятся нам в этом гиде.

*   **Ядро (Kernel):** Это «мозг» и «сердце» операционной системы. Оно управляет процессором, памятью и устройствами. Обычный пользователь не видит ядро, но именно оно решает, какая программа получит ресурсы в данный момент.
*   **ISO-образ:** Это цифровой слепок компакт-диска. Почему не `.exe`? Потому что `.exe` — это программа для Windows. А ISO — это файл, который содержит в себе всю файловую систему установочного диска, включая загрузочные сектора. Мы «вставим» его в виртуальный привод нашей машины.
*   **GRUB:** Загрузчик системы. Это первая программа, которая запускается при включении «компьютера». Именно она спрашивает вас: «Какую систему запустить?» (если их несколько) и передает управление ядру Linux.
*   **Репозиторий (Repository):** Это аналог App Store или Google Play в мире Linux. Огромное хранилище проверенного программного обеспечения, откуда мы будем скачивать инструменты одной командой.
*   **Зависимости (Dependencies):** Представьте, что вы строите дом. Чтобы положить крышу, вам нужен молоток и гвозди. Крыша «зависит» от молотка. В Linux программы часто используют функции друг друга. Если программе А для работы нужен компонент Б, то Б является её зависимостью.
*   **DHCP:** Система автоматической раздачи настроек сети. Ваш домашний роутер — это DHCP-сервер. Когда вы подключаете телефон к Wi-Fi, он получает адрес именно от него.
*   **Root / Суперпользователь:** Это «Администратор» на стероидах. Пользователь с неограниченными правами. В Linux принято работать под обычным пользователем, а права Root использовать только тогда, когда это действительно нужно (через команду `sudo`).

## Зачем Linux в DevOps?

Если вы планируете карьеру в DevOps, Linux станет вашей основной средой обитания. Почему не Windows Server? Почему не что-то другое? Давайте разберем основные причины, которые сделали Linux стандартом де-факто в индустрии.

### Ubuntu Server vs Desktop: В чем разница?
Перед тем как перейти к практике, ответим на важный вопрос: чем серверная версия отличается от той, что стоит у вас на ноутбуке?
1. **Отсутствие GUI:** В Ubuntu Server нет графической оболочки (окон, иконок, рабочего стола). Это экономит 1-2 ГБ оперативной памяти и делает систему более защищенной.
2. **Набор софта:** Здесь нет браузера Firefox, LibreOffice или медиаплееров. Зато «из коробки» идут инструменты для работы сети, SSH-сервер и системные утилиты.
3. **Режим работы:** Сервер рассчитан на работу 24/7 без участия человека. Все настройки выполняются через текстовые файлы конфигурации.

### Почему Linux — стандарт индустрии?
1. **Доминирование и универсальность:** AWS, Google Cloud, Azure — все они построены на Linux. Контейнеры (Docker) и Kubernetes — нативно живут в мире Linux.
2. **Философия Open Source:** Вы не платите за лицензии, имеете полный доступ к коду и поддержку огромного сообщества.
3. **Эффективность ресурсов:** Отсутствие графики позволяет отдавать всю мощность процессора и памяти вашим приложениям.
4. **Автоматизация через CLI:** Командная строка позволяет управлять тысячами серверов одновременно с помощью скриптов.

Мы выбрали **Ubuntu Server 22.04 LTS**. Аббревиатура LTS (Long Term Support) гарантирует поддержку и обновления безопасности в течение минимум 5 лет.

## Подготовка: Виртуальное «железо» и BIOS

Для обучения нам не нужно покупать отдельный компьютер. Мы воспользуемся **VirtualBox**.

### Шаг 0: Pre-flight check (Важно!)
Убедитесь, что ваш компьютер готов к виртуализации:
1.  **Проверка в Windows:** `Ctrl + Shift + Esc` -> Производительность -> ЦП. Пункт **Виртуализация** должен быть «Включено».
2.  **Если выключено:** Зайдите в BIOS/UEFI и включите **Intel Virtualization Technology** или **AMD-V**.

### Шаг 1: Установка VirtualBox
1. Скачайте инсталлятор с [virtualbox.org](https://www.virtualbox.org/).
2. Установите его, соглашаясь со всеми настройками по умолчанию.

### Шаг 2: Скачивание Ubuntu ISO
1. Перейдите на [ubuntu.com](https://ubuntu.com/download/server).
2. Скачайте **Ubuntu Server 22.04 LTS**.

**Рекомендуемые ресурсы для ВМ:**

| Ресурс | Минимум | Рекомендуется |
|--------|---------|---------------|
| RAM | 2 ГБ | 4 ГБ |
| CPU | 1 ядро | 2 ядра |
| Диск | 15 ГБ | 25-30 ГБ |
| Сеть | NAT | Bridged Adapter |

## Создание виртуальной машины

В VirtualBox нажмите «Создать» (New):
1. **Имя:** `ubuntu-server-2204`. Выберите скачанный ISO. Поставьте галочку **Skip Unattended Installation**.
2. **Hardware:** Минимум **2048 MB RAM** и **2 CPU**.
3. **Hard disk:** **20-25 GB**, тип VDI, динамический.
4. **Сеть:** В настройках выберите **Сетевой мост (Bridged Adapter)**. Это позволит серверу получить IP от вашего роутера и быть доступным в домашней сети.

---

## Установка системы: Шаг за шагом

1. **Запуск:** Выберите `Try or Install Ubuntu Server`.
2. **Язык:** Выбирайте **English**. Это стандарт в IT.
3. **Сеть:** Убедитесь, что получен IP по DHCP.
4. **Разметка диска:** Выберите `Use an entire disk`. **Снимите галочку** с `Set up this disk as an LVM group` для простоты на первом этапе.
5. **Профиль:** Создайте пользователя (например, `devopsuser`) и придумайте пароль. **Запишите их!**
6. **SSH:** Обязательно выберите **[X] Install OpenSSH server**.
7. **Завершение:** После установки выберите `Reboot Now`.

## Первичная настройка после установки

После перезагрузки введите логин и пароль. Помните: при вводе пароля в Linux символы не отображаются.

### 1. Обновление системы: Update vs Upgrade
```bash
# Обновляем списки доступных пакетов (каталог)
sudo apt update

# Устанавливаем новые версии программ
sudo apt upgrade -y
```

### 2. Настройка идентификации и окружения
После обновления важно задать серверу правильные параметры.

*   **Установка имени хоста (Hostname):**
    ```bash
    sudo hostnamectl set-hostname devops-server
    ```
    *Зачем:* Уникальное имя помогает не запутаться в сети из множества серверов.

*   **Настройка часового пояса:**
    ```bash
    sudo timedatectl set-timezone Europe/Moscow
    ```
    *Зачем:* Точное время критично для анализа логов и работы запланированных задач (cron). Список поясов: `timedatectl list-timezones`.

*   **Конфигурация локали:**
    ```bash
    sudo localectl set-locale LANG=en_US.UTF-8
    ```
    *Зачем:* Английский язык в консоли — стандарт. Ошибки легче гуглить, нет проблем с кодировкой.

### 3. Проверка и запуск SSH
Если вы забыли установить SSH при инсталляции:
```bash
sudo apt install -y openssh-server
sudo systemctl enable --now ssh
```
Команда `enable --now` запускает службу сейчас и добавляет её в автозагрузку.

### 4. Проверка конфигурации
Убедитесь, что все настройки применены корректно:

```bash
# Проверка имени хоста
hostnamectl

# Проверка часового пояса
timedatectl

# Проверка локали
localectl status

# Проверка IP-адреса
ip a
```

## Подключение по SSH: Комфортная работа

Работать в терминале через SSH гораздо удобнее (работает копипаст, прокрутка).

### 1. Узнаем IP-адрес
```bash
ip a
```
Ищите адрес в разделе `enp0s3` после слова `inet` (например, `192.168.1.15`).

### 2. Подключение из Windows
Откройте PowerShell или Windows Terminal:
```bash
ssh devopsuser@192.168.1.15
```
Если у вас старая Windows, используйте **PuTTY**.

### 3. Настройка фаервола (UFW)
```bash
sudo ufw allow ssh
sudo ufw enable
```

## Что делать, если что-то пошло не так (Troubleshooting)

### 1. Ошибка «Virtualization is disabled in BIOS»
*   **Симптом:** Виртуальная машина не запускается.
*   **Решение:** Перезагрузите компьютер, зайдите в BIOS (F2/Del) и включите **Intel Virtualization Technology** или **SVM Mode**.

### 2. Не удается подключиться по SSH
*   **Проверьте IP:** Убедитесь, что вы используете правильный адрес из `ip a`.
*   **Проверьте статус:** Введите `sudo systemctl status ssh`. Если служба выключена, запустите её: `sudo systemctl start ssh`.
*   **Сеть:** Убедитесь, что в настройках VirtualBox выбран «Сетевой мост».

### 3. Нет интернета в виртуалке
*   **Симптом:** `apt update` выдает ошибки.
*   **Решение:** Попробуйте сменить тип адаптера в VirtualBox на **NAT**. Это самый простой способ получить интернет, если «Сетевой мост» блокируется вашим провайдером или роутером.
