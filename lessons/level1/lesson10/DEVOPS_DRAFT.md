# Урок 10 — Написание простых Bash-скриптов (Technical Base)

## Введение
Bash (Bourne Again SHell) — это стандартный командный интерпретатор в большинстве дистрибутивов Linux. Для DevOps-инженера Bash является "швейцарским ножом" для автоматизации рутины: от простых бэкапов до сложных CI/CD пайплайнов.

---

## 1. Особенности Bash 5.x
Версия 5.x принесла несколько важных улучшений, которые делают скрипты мощнее и надежнее:
*   **Ассоциативные массивы:** Улучшенная поддержка `declare -A`.
*   **Переменная `$EPOCHSECONDS`:** Позволяет получать текущее время в секундах без вызова внешней команды `date`.
*   **`wait -n`:** Ожидание завершения любого из фоновых процессов.
*   **Улучшенная обработка `$BASH_LOADABLES`:** Динамическая загрузка встроенных команд.

---

## 2. Фундамент надежного скрипта: `set -euo pipefail`
Это "золотой стандарт" для любого production-скрипта. Без этой строки скрипт может продолжить выполнение при ошибке, что приведет к непредсказуемым последствиям.

*   **`set -e` (errexit):** Скрипт немедленно завершается, если любая команда возвращает ненулевой статус выхода (код ошибки).
*   **`set -u` (nounset):** Вызывает ошибку при попытке использования необъявленной переменной. Это предотвращает случайное удаление корня при опечатке в пути (например, `rm -rf $DEL_PATH/` если `$DEL_PATH` пуста).
*   **`set -o pipefail`:** Если команда в цепочке pipe (`|`) завершится с ошибкой, вся цепочка вернет код ошибки (по умолчанию возвращается код последней команды).

## 3. Инструменты логирования и перехвата ошибок

### `trap`
Позволяет выполнять действия при получении сигналов (например, `SIGINT`, `EXIT`). Идеально для очистки временных файлов.
```bash
cleanup() {
    rm -rf "$TEMP_DIR"
    echo "Temporary files removed."
}
trap cleanup EXIT
```

### `logger`
Отправляет сообщения в системный лог (`syslog` или `journald`). Это стандарт для серверных скриптов.
```bash
logger -t "BACKUP_SCRIPT" "Backup started successfully"
```

### `tee`
Позволяет одновременно выводить данные в терминал и записывать их в файл.
```bash
echo "Process started" | tee -a /var/log/my_script.log
```

## 4. Переменные, Аргументы и Ввод данных

### Аргументы командной строки
В Bash есть специальные переменные для работы с аргументами:
*   `$0` — Имя самого скрипта.
*   `$1, $2...$9` — Позиционные аргументы.
*   `$#` — Количество переданных аргументов.
*   `$@` — Все аргументы как отдельные слова.
*   `$?` — Код выхода последней выполненной команды.

### Разбор опций (`getopts`)
Для более гибкого управления (напр. `-v`, `-p port`) используйте `getopts`:
```bash
while getopts "h:v" opt; do
  case $opt in
    h) echo "Help menu"; exit 0 ;;
    v) echo "Verbose mode enabled";;
    *) echo "Unknown option"; exit 1 ;;
  esac
done
```

### Ввод пользователя (`read`)
Иногда скрипту нужно интерактивное подтверждение:
```bash
read -p "Вы уверены, что хотите удалить базу данных? (y/n): " confirm
if [[ "$confirm" == "y" ]]; then
    echo "Удаление..."
fi
```

## 5. Циклы и Условия

### Циклы
*   **`for`**: Идеален для перебора списков (файлов, серверов).
    ```bash
    for server in srv1 srv2 srv3; do
        ssh "$server" "uptime"
    done
    ```
*   **`while`**: Выполняется, пока условие истинно.
    ```bash
    count=1
    while [ $count -le 5 ]; do
        echo "Попытка #$count"
        ((count++))
    done
    ```
*   **`until`**: Выполняется, пока условие ложно (до тех пор, пока не станет истинным).

## 6. Скрипт №1: Ротируемый бэкап (Backup Script)
Этот скрипт архивирует папку, использует аргументы командной строки и проверяет наличие инструментов.

```bash
#!/bin/bash

# Безопасный режим
set -euo pipefail

# Конфигурация
BACKUP_SRC="${1:-/var/www/html}"    # Источник из $1 или значение по умолчанию
BACKUP_DEST="/mnt/backups"
DATE=$(date +%Y-%m-%d_%H%M%S)
LOG_FILE="/var/log/backup_script.log"

# Функция логирования
log_message() {
    local MSG="$1"
    echo "[$DATE] $MSG" | tee -a "$LOG_FILE"
    logger -t "WEB_BACKUP" "$MSG"
}

# Проверка прав (должен быть root)
if [[ $EUID -ne 0 ]]; then
   echo "Этот скрипт должен быть запущен от имени root"
   exit 1
fi

# Проверка существования директорий
mkdir -p "$BACKUP_DEST"

log_message "Начало бэкапа директории $BACKUP_SRC"

# Создание архива
tar -czf "$BACKUP_DEST/backup_$DATE.tar.gz" "$BACKUP_SRC" 2>>"$LOG_FILE"

log_message "Бэкап успешно создан: $BACKUP_DEST/backup_$DATE.tar.gz"

# Ротация: удаляем файлы старше 7 дней
find "$BACKUP_DEST" -type f -name "*.tar.gz" -mtime +7 -delete
log_message "Старые бэкапы удалены (ротация 7 дней)."
```

## 5. Скрипт №2: Мониторинг ресурсов и уведомление (Health Check)
Скрипт проверяет место на диске и использование оперативной памяти.

```bash
#!/bin/bash

set -euo pipefail

THRESHOLD_DISK=80  # Порог в %
THRESHOLD_MEM=90   # Порог в %

check_disk() {
    local usage
    usage=$(df / | awk 'NR==2 {print $5}' | sed 's/%//')
    if [[ "$usage" -gt "$THRESHOLD_DISK" ]]; then
        echo "WARNING: Диск заполнен на $usage%!" | logger -p user.warn
        return 1
    fi
    return 0
}

check_mem() {
    local mem_usage
    mem_usage=$(free | grep Mem | awk '{print $3/$2 * 100.0}' | cut -d. -f1)
    if [[ "$mem_usage" -gt "$THRESHOLD_MEM" ]]; then
        echo "CRITICAL: Память заполнена на $mem_usage%!" | logger -p user.crit
        return 1
    fi
    return 0
}

# Основной цикл
echo "--- System Health Check ---"
check_disk || echo "Проблема с диском"
check_mem || echo "Проблема с памятью"

if [[ $? -eq 0 ]]; then
    echo "Все системы в норме."
fi
```

## 6. Автоматизация через `cron`
Чтобы скрипт запускался автоматически, используем `crontab -e`.

Пример записи для бэкапа (каждый день в 03:00):
```cron
0 3 * * * /usr/local/bin/backup.sh /home/user/data >> /var/log/cron_backup.log 2>&1
```

## 7. Best Practices & Anti-patterns

### ✅ Best Practices:
1.  **Shebang:** Всегда используйте `#!/bin/bash` (или `#!/usr/bin/env bash`).
2.  **Permissions:** Устанавливайте минимально необходимые права (`chmod 700` для скриптов с паролями).
3.  **Комментарии:** Описывайте логику функций и назначение переменных.
4.  **Переменные в кавычках:** Пишите `"$VAR"`, а не `$VAR`, чтобы избежать проблем с пробелами в путях.
5.  **Long options:** Используйте `rm --recursive` вместо `rm -r` для читаемости в скриптах.

### ❌ Anti-patterns:
1.  **Hardcoding:** Не зашивайте пароли и пути прямо в код. Используйте переменные окружения или конфиг-файлы.
2.  **Игнорирование ошибок:** Не оставляйте скрипт без `set -e`.
3.  **Использование `ls` для парсинга:** Для работы с файлами используйте `find` или `globbing` (напр. `for file in *.jpg`). `ls` может сломаться на именах с переносами строк.
4.  **Копипаст:** Если код повторяется более 2 раз — выносите его в функцию.

## 8. Отладка (Debugging)
Самый простой и эффективный способ дебага — запуск с флагом `-x`:
```bash
bash -x ./my_script.sh
```
Это выведет каждую команду перед её выполнением, подставляя значения переменных.
