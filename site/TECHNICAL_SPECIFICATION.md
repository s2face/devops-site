## 1. Общее описание проекта

### 1.1. Назначение
Разработка веб-сайта для публикации и изучения учебных материалов по DevOps. Сайт должен предоставлять структурированный доступ к статьям курса, обеспечивать навигацию между уроками и отслеживание прогресса обучения.

### 1.2. Целевая аудитория
- Начинающие DevOps-инженеры (Level 1)
- Инженеры с опытом, повышающие квалификацию (Level 2, Level 3)
- Самообучающиеся специалисты IT

### 1.3. Источники контента
Контент берётся из директории `lessons/`:
```
lessons/
├── level1/    — 15 готовых уроков (ARTICLE.md)
├── level2/    — 15 уроков в разработке (task.md)
└── level3/    — 15 уроков в разработке (task.md)
```

## 4. Формат контента (КРИТИЧНО)

### 4.1. Frontmatter для ARTICLE.md

Каждый файл `ARTICLE.md` **должен** содержать следующий frontmatter:

```yaml
---
lesson:
  number: 1
  title: "Введение в DevOps"
  slug: "vvedenie-v-devops"
  level: 1
  status: "ready"  # ready | draft | todo
  
navigation:
  previous: null           # null или slug предыдущего урока
  next: "ubuntu-server"    # slug следующего урока
  prerequisites: []        # массив slug пререквизитов
  
meta:
  description: "Первый урок курса DevOps. Введение в культуру DevOps, метрики DORA."
  keywords: ["DevOps", "DORA", "KPI", "CI/CD"]
  updated: "2026-03-29"
---
```

### 4.2. Структура тела статьи

Тело статьи **должно** содержать следующие секции (порядок обязателен):

```markdown
# Заголовок урока (дублирует frontmatter.title)

## Введение
...

## Основная часть
...

---

## Ключевые выводы урока

1. Первый вывод
2. Второй вывод
3. ...

---

## Глоссарий терминов урока

| Термин | Расшифровка | Простое объяснение |
|--------|-------------|-------------------|
| KPI | Key Performance Indicators | Ключевые показатели эффективности |
| DORA | DevOps Research and Assessment | Метрики эффективности команды |

---

## Чек-лист готовности урока

- [ ] Навык 1
- [ ] Навык 2
- [ ] ...
```

### 4.3. Convention по именованию секций

| Секция | Обязательные заголовки (точно) | Альтернативы |
|--------|-------------------------------|--------------|
| Введение | `## Введение` | `## Introduction` |
| Ключевые выводы | `## Ключевые выводы урока` | `## Key Takeaways` |
| Глоссарий | `## Глоссарий терминов урока` | `## Глоссарий`, `## Glossary` |
| Чек-лист | `## Чек-лист готовности урока` | `## Чек-лист`, `## Checklist` |

> **Парсер** должен поддерживать оба варианта (русский + английский).

## 5. Механизм парсинга контента

### 5.1. Архитектура парсера

```
ARTICLE.md
    │
    ▼
┌─────────────────┐
│  gray-matter    │ → frontmatter (Lesson metadata)
└─────────────────┘
    │
    ▼
┌─────────────────┐
│  remark-parse   │ → AST дерева Markdown
└─────────────────┘
    │
    ▼
┌─────────────────┐
│  parser.ts      │ → Извлечение секций по заголовкам
└─────────────────┘
    │
    ├── takeaways: string[]
    ├── glossary: Term[]
    └── checklist: string[]
```

### 5.2. Алгоритм извлечения данных

#### Ключевые выводы:
1. Найти заголовок `## Ключевые выводы урока` или `## Key Takeaways`
2. Извлечь нумерованный список до следующего заголовка `##` или конца документа
3. Очистить от Markdown-разметки (оставить текст)

#### Глоссарий:
1. Найти заголовок `## Глоссарий терминов урока` или `## Glossary`
2. Извлечь Markdown-таблицу
3. Распарсить в массив объектов `{ term, expansion, explanation }`

#### Чек-лист:
1. Найти заголовок `## Чек-лист готовности урока` или `## Checklist`
2. Извлечь маркированный список с чекбоксами (`- [ ]` или `- [x]`)
3. Очистить от синтаксиса чекбоксов

### 5.3. Обработка ошибок парсинга

| Ошибка | Действие |
|--------|----------|
| Секция не найдена | Пустой массив, warning в консоль сборки |
| Таблица глоссария невалидна | Пропустить строку, warning с номером строки |
| Frontmatter отсутствует | Ошибка сборки, урок не включается |
| Slug дублируется | Ошибка сборки, список дубликатов |

### 5.4. TypeScript интерфейсы

```typescript
// src/types/index.ts

export type LessonStatus = 'ready' | 'draft' | 'todo';
export type LessonLevel = 1 | 2 | 3;

export interface LessonFrontmatter {
  lesson: {
    number: number;
    title: string;
    slug: string;
    level: LessonLevel;
    status: LessonStatus;
  };
  navigation: {
    previous: string | null;
    next: string | null;
    prerequisites?: string[];
  };
  meta: {
    description: string;
    keywords: string[];
    updated: string;  // ISO 8601
  };
}

export interface Term {
  term: string;
  expansion: string;
  explanation: string;
}

export interface Lesson {
  frontmatter: LessonFrontmatter;
  content: string;      // Основное содержимое (без секций выводов/глоссария/чеклиста)
  takeaways: string[];  // Извлечённые выводы
  glossary: Term[];     // Извлечённый глоссарий
  checklist: string[];  // Извлечённый чек-лист
}

export interface LessonProgress {
  lessonSlug: string;
  checklistState: boolean[];  // Состояние каждого пункта
  completedAt?: string;       // Дата завершения (если все пункты отмечены)
  version: string;            // Версия контента для инвалидации
}
```

### 6.2. Версионирование контента

| Событие | Действие |
|---------|----------|
| Обновление ARTICLE.md | Увеличение `meta.updated`, генерация хэша контента |
| Изменение структуры | Увеличение мажорной версии формата |
| Прогресс пользователя | Сохранение `version` в localStorage, сброс при несовпадении |

**Алгоритм инвалидации:**
1. При загрузке урока сравнивать `version` в localStorage с текущим хэшем контента
2. Если не совпадает — сбросить прогресс, показать уведомление:
   > «Содержимое урока обновлено. Ваш прогресс сброшен.»

### 6.3. Cross-references

**Формат в frontmatter:**
```yaml
navigation:
  prerequisites:
    - "osnovy-bash"
    - "ubuntu-server"
  related:
    - level: 2
      lesson: "kubernetes-pods"
      label: "Продолжение: Kubernetes"
```

**Рендеринг:**
- Блок «Пререквизиты» перед содержанием (если есть)
- Блок «Смотрите также» после содержания

## 15. Риски и митигация

| Риск | Вероятность | Влияние | Митигация |
|------|-------------|---------|-----------|
| **Парсинг ARTICLE.md сложен** | Средняя | Высокое | Чёткий формат frontmatter + секций, тесты парсера |
| **contentlayer устарел** | Высокая | Среднее | Использовать `gray-matter` + `remark` (активные проекты) |
| **Symlink не работает на Windows** | Высокая | Высокое | Скрипт синхронизации вместо symlink |
| **Отсутствие контента Level 2/3** | Высокая | Низкое | Статус «В разработке», заглушки |
| **XSS при рендеринге Markdown** | Низкая | Высокое | Санитизация через `rehype-sanitize` |
| **SEO при CSR чек-листов** | Средняя | Среднее | SSR рендеринг чек-листов (состояние по умолчанию) |
| **Vercel free tier недостаточен** | Низкая | Низкое | Альтернатива: Netlify, Cloudflare Pages |
| **Потеря прогресса (localStorage)** | Средняя | Низкое | Версионирование, уведомление при сбросе |
| **Производительность на мобильных** | Средняя | Среднее | Оптимизация изображений, код-сплиттинг |
